03 ·

Color

Neutral scale is Tailwind zinc. Accents are tuned per-theme so every color clears WCAG AA (≥ 4.5:1) on white. Mint (#2dd4bf) is the only brand hue. Success (emerald #10b981) is deliberately a different green so a positive delta never reads as "this is clickable."

The Four Laws of Color
Law 01
Mint for actions.
Brand, primary CTA, active/selected state, toggles, focus rings, tier badges — all mint. If it's clickable or currently-on, it's mint. Never blue, violet, or indigo here.
#2dd4bf dark · #0d9488 light
Law 02
Blue for data.
Clicks, sessions, impressions, links, progress bars, “Needs Review” — all blue. Blue is read-only. It labels a number; it never invites a click.
text-blue-400 · bg-blue-500/10
Law 03
Emerald for success.
Positive deltas, completion checks, “Live” pills, approvals, “Win” badges. Emerald is never clickable — if you mix it with a button, you've created a bug.
#10b981 dark · #047857 light
Law 04
Purple for admin AI only.
Reserved for AdminChat and admin-only AI features. Purple never appears in any client-facing view — it signals agency-side intelligence, not brand.

06 ·

Data visualization

Charts carry more weight than any other surface — they are the product’s argument. This chapter governs what a chart looks like, what it’s allowed to do, and how it behaves when data is late, absent, or broken. Every rule exists because the alternative was seen and rejected.

06.0 · The base chart — TrendChart, dual series
Clicks & impressions · 28 days
Mar 2 – Mar 29, 2026
Clicks Impressions
Stroke 1.5px · gradient .18 → 0 · vector-effect: non-scaling-stroke · sine/cosine-shaped data so every inflection is a real inflection
This is the reference. Every chart in the atlas, the annotations overlay, the receipt treatment, and the anatomy diagram pulls its stroke weight, gradient ramp, grid dashing, and axis type from this one exhibit. If the two don’t match, this one wins.
Dual-series is canonical. Clicks (Blue 400) vs. impressions (Teal) — two different metrics on a dual axis is the one place we allow a second hue. Use it only when units differ; for same-unit comparisons (this period vs. prev, mobile vs. desktop) use two values of Blue. See §06.1.
06.1 · Laws of color — one color per meaning, not per chart
01
Single series
Blue 400. That’s it.
Clicks
Use for: any standalone metric. One number, one hue, no ambiguity.
02
Two series, same unit
Same hue, different value. Blue 400 + Blue 600.
Mobile Desktop
Use for: mobile vs. desktop · cohort vs. cohort · any two slices of the same metric.
03
Two different metrics
ONLY WHEN UNITS DIFFER
Blue 400 + Teal. No other second-hue is permitted.
Clicks Impressions
Use for: clicks vs. impressions · sessions vs. revenue · anything requiring a dual axis. This is the base-chart treatment in §06.0.
DON’T Three or more colors.
If you need to compare three or more series, you’re drawing the wrong chart. Switch to small-multiples (see atlas #5) — one tile per series, all Blue 400. A tangled rainbow line is unreadable; six small charts are scannable.
DON’T Second hue for the same metric.
“This period vs. previous” is not two metrics — it’s one metric at two times. Use the ghost treatment (Blue 400 solid + Blue 400 at 28%), never teal for prev. Teal earns its keep by signaling a different unit; use it anywhere else and the signal breaks.
The other three laws
Ghost = previous period.
When comparing timeframes, the current period is solid Blue 400. The previous period is the same blue at 28% opacity, no dots, no label in the legend — just a whisper behind the real line. Any other comparison chrome is invented.
Emerald / Amber / Red are earned.
Score-colors appear when a number crosses a line — never as decoration. 80+ emerald, 60+ amber, below red. A bar chart where every bar is emerald because “it looked nice” is the failure mode this rule prevents.
The chart is not the story.
Every chart ships with a one-line verdict above or beside it — “Clicks up 12% this week” — not just axes and numbers. If a chart can’t be captioned, it doesn’t belong.
Teal is a budget, not a palette.
Teal can appear once per chart, once per page, only when a legitimately-different metric demands a second axis. If you find teal showing up on the same page twice, the page is trying to say too much.
The Blue scale — five steps, one hue
Use Blue 400 for a single series. Use Blue 400 + Blue 600 for two same-unit series (case 02 above). Use 200 → 600 for stacked bars, grouped bars, or small-multiple series where order matters. The only permitted non-blue second hue is Teal, and only when the second series is a genuinely different unit (case 03).

Dynamic axis assignment — how the chart picks left vs. right
Every active line has a yAxisId hint (left|right). The chart doesn’t trust the hint blindly — it runs assignAxes() over the real data and can override. Four cases cover every permutation:
01 · SINGLE LINE Left
Always left. Right axis doesn’t render at all — hasRightAxis is false.
02 · SAME HINT · CLOSE SCALE Share left
Both lines hint left; peak ratio < 10×. They share the axis — this is mobile vs. desktop.
03 · SAME HINT · 10× RATIO Auto-split
Peak ratio ≥ 10× — the smaller metric gets moved to right so it doesn’t flatline. Clicks (1K) + impressions (180K).
04 · RATE-VS-VOLUME CONFLICT Guarded
A volume metric would move right, but the right axis already holds a rate (CTR, position). Moving would break both. Rule 13 — rate wins; volume stays compressed on left.
Ratio threshold is hard-coded: ≥ 10. Below that, metrics share. assignAxes() also color-codes each axis fill via activeLines.find(l => axisAssignments.get(l.key) === ‘left’)?.color.

Motion

Three durations. Every animation in the system picks one of the three — no in-between values. Instant for state response, Standard for UI changes, Deliberate for the rare moments that earn the spotlight. Ease is uniformly ease-out unless noted.

01 · Instant
100ms
Hover, focus, toggle, press. The UI should feel like it's responding to you, not performing for you.
02 · Standard
240ms
Card entries, modal open/close, tab switch, toast, number fade-in, stagger step. The workhorse — 90% of animations land here.
03 · Deliberate
600ms
Moments of consequence: ring charge-up, scanner reveal, celebration glow. Use sparingly — once per screen, at most.

Accessibility contract
Focus trap (Modal)Tab cycles through focusables inside the dialog; Shift+Tab wraps the opposite direction. On close, focus restores to the element that opened the modal.
Escape everywhereEscape closes Modals, Popovers, and hides Tooltips. Escape inside a text field still takes precedence because we never trap Escape on focused inputs unless the overlay is open.
