## 4. What is still open

### Launch-relevant (decide before/at promote)

- **B3 re-arm hazard (the one I'd lose sleep over).** The same-tick auto-send coupling is dormant, not removed. *Nothing in code* forces a decoupled-tick + operator-veto window to exist before the first auto-send fires — only the flag's `removalCondition` prose and operator discipline. Customer Success flagged the sharpest version: the admin feature-flag-toggle UI on your own roadmap could flip `strategy-trust-ladder-autosend` ON per-workspace through a config screen — re-arming the exact round-1 blocker *without a code change*. There is no test asserting "flag-on requires the veto path to exist first."
- **Trial-window identical-greenlight (newly surfaced by Director of Revenue).** `computeEffectiveTier` promotes Free→Growth for an active trial, so the B1 gate does *not* fire for trial clients. A 14-day trial client can stack ~10 monetizable content requests (~$2,500–8,000 of queued billable scope) at zero charge and let the trial lapse. Intentional (trials demonstrate value), but there's no per-trial greenlight cap. **Owner decision, not a silent default.**
- **The "no charge" promise is now load-bearing copy.** Both the confirm dialog and success toast promise "your strategist will scope and confirm before any work or charge." Director of Revenue did *not* verify a downstream operator-side gate enforcing "confirm before charge." If fulfillment ever auto-progresses a request into billable work without that step, it becomes a chargeback/dispute liability.
- **Ephemeral staged-set footgun (raised by 3 panelists).** Staging lives in component-local `useToggleSet` — wiped on tab unmount / navigation / refresh with no unsaved-work warning. An operator who stages 8 moves, gets distracted, and navigates away loses the staging silently. The per-row-send model didn't carry this; the (correct) single-commit model introduced it.

### Deferred fast-follows (honestly labeled, not silently dropped)

- **External product name + strategist identity (D11 / FF#4).** Zero client-facing product name ships; the only "The Issue" string is an internal comment. Head of Marketing's owned blocker is **unresolved by design** — "lock before any external demo/sale." Internally launch-ready, commercially un-sellable on day one. The "Curated by your strategist" byline shipped (a thin, cloneable signal); the cockpit — the real sales asset — is still invisible to buyers.
- **Growth à-la-carte charge (FF#2).** Launch model is tier-as-gate with no incremental charge; `priced` mode exists in the union but is never emitted. Growth clients greenlight unlimited monetizable moves at flat MRR — a knowing margin decision, but unbounded fulfillment cost at scale.
- **Outcome/export one-pager + funnel bridge (FF#10/FF3).** The budget-defense and CMO-attribution artifact. Both customers' decisive-tier ask. Period-over-period is reserved for this export; it doesn't exist yet.
- **Portfolio triage + greenlight inbox (FF#7/FF#1).** `POV_UNCHANGED` is computed but never rolled up cross-account. Customer Success: launch-safe for ≤5–8 accounts; gates any sales push beyond that. Not a launch blocker for a solo operator today.
