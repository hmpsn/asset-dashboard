# Strategy v3 — Client Overview Version Compare (gated vs ungated)

**Date:** 2026-06-18  ·  **Method:** 2 grounded catalogers (ungated current vs strategy-command-center gated) + synthesis, via Workflow `client-overview-version-compare` (wrrrh9wfd).

## Engagement verdict — why the ungated version feels more engaging

The ungated version feels more engaging for three transferable reasons, all grounded in the source I read. (1) NARRATION OVER DISPLAY — `OverviewTab.dynamicSubtitle` (lines 143-159) turns the single biggest number into a sentence ("Traffic is up 14% — here's what's driving it"), and the briefing body opens with `WeeklyOpener`+`DateLine`+`IssueSummaryLine` (InsightsBriefingPage:237-239) framing the page as "YOUR weekly issue." The gated version opens cold on a ring + a static `CompactStatBar` (OrientHeader:50-70) — no greeting, no story, no "letter from the editor." (2) PROOF THE FEE IS EARNED — `WinsSurface` renders dated, dollar-quantified per-win rows ("Published new post · Strong win · +12.4 (+38%) · ≈$340 in added traffic value", WinsSurface:74-95), `OutcomeSummary` shows measured win rates, and `MonthlyDigest` reads like a report written for the client. The gated version is a keyword/page WORKBENCH (tables, priorities forms, declined lists) — valuable management surface, but it states no results. (3) OPINION — the ungated `#1 Priority` card (OverviewTab:290-345) picks ONE thing and explains WHY with relative contribution bars ("Why this is your top priority"), and contextual CTA banners push a single next action. The gated version's only opinionated copy is the OrientHeader `verdict()` helper (OrientHeader:19-24) and the content/page rec reason copy — good, but isolated. TRANSFERABLE INGREDIENTS to bake into v3: (a) the dynamicSubtitle/issue-masthead "narrate the biggest number" engine for the Briefing opener; (b) the per-win dated/$ proof row as the v3 hero-win element; (c) the "why this is #1" contribution-bar explanation pattern on curated rec cards; (d) the OrientHeader `verdict()`/`signed()`/`positionSub()` plain-English score helpers for the Layer-1 score line. The gated version's ONE thing the ungated does worse — a CTR-WEIGHTED visibility score with narrative verdict (vs a generic composite health ring) — is the headline v3 should adopt.

## Composition recommendation (the answer to "what to keep from each")

Compose the v3 curated overview as: BRIEFING OPENER drawn from the UNGATED briefing body — the WeeklyOpener+DateLine+IssueSummaryLine masthead trio plus the dynamicSubtitle prose engine, with the GATED StrategyRefreshSummarySection's New/Moved/Retired framing folded in as change-context. LAYER-1 drawn primarily from the GATED version — StrategyClientOrientHeader's CTR-weighted visibility score + verdict() helpers as the single amber ring (retire both ungated rings AND the gated Snapshot /100), grafting on HealthScoreCard's expandable 'what makes up this score' disclosure; pair it with the UNGATED WinsSurface as the hero-win proof strip and OutcomeSummary as the results surface; use the briefing PulseStrip's data model for the lean stats line. LAYER-3 (the ONE curated recs layer) built from the UNGATED RecommendedForYou (split paid/free CTAs — the canonical card) and the GATED ContentGapCard + buildGrowthOpportunityPages reason copy as co-feeders, headed by ActionQueueStrip (standardized over the classic banner, + a new progress bar) and carrying the #1-Priority 'why this is your top priority' contribution-bar explanation onto each card. Everything else absorbs or retires: from ungated RETIRE the contextual CTA banner, PredictionShowcaseCard, IntelligenceSummaryCard, the Content Opportunities sidebar preview, the duplicate StatCard grid, and the legacy Recent Work timeline; harvest InsightsDigest's card patterns but cut its volume. From gated RETIRE the interior TabBar IA, StrategyNextStepsSection, StrategySnapshotSection's headline score, and leave all the workbench surfaces (business priorities, keyword tables, page-keyword map, declined, feedback summary, requested-trend) on the standalone strategy tab — v3 links into them, never embeds them. DONE-STATE drawn from the UNGATED empty-setup card + FreeTierUpgradeCTA + AgencyWorkFeed's in-progress block (promote it from its flag) to compose the v3 quiet-month/done screen. Net: the visibility SCORE and the rec-card intelligence come from the gated version; the narrative DNA, the proof surfaces, and the canonical rec card come from the ungated version.

## v3 gaps surfaced (net-new the mockup needs)

1. MISSING DATA for the hero-win attribution chain. The v3 mockup's signature hero-win line is 'We recommended it → you approved it → here's the result' (mockup line 102). I verified OutcomeWinEntry (shared/types/outcome-tracking.ts:266-296) carries a `recommendation` source title (line 272) and `attributedValue` (276) but has NO approval-step / 'you approved it' field. The recommend→APPROVE→result chain is a v3-net-new narrative device that neither WinsSurface nor the outcome data shape currently supports. v3 either needs a new field linking the win back to the client's approval event, or must soften the chain to the two steps the data can prove (we recommended → result).

2. MISSING the segmented progress bar. The v3 mockup's '2 need your decision' header is paired with a 3-segment progress bar (approved/in-discussion/remaining, mockup line 112). Neither ActionQueueStrip nor the classic Action-needed banner renders a progress bar today — both only show count chips. This is net-new UI to build for the v3 decisions header.

3. MISSING an explicit in-discussion / live-reply rec state. The v3 mockup has an 'In discussion' rec card with a live inline reply composer (mockup lines 131-136). The existing rec surfaces (RecommendedForYou, ContentGapCard) reflect request STATUS but don't render a conversation thread + reply box inline on the overview. This conversational rec state is new.

4. SCORE SOURCE NOT YET DECIDED in v3. The mockup ring is 'visibility' framed (64, amber, '+5 this month', 'your search visibility is building') which matches the gated OrientHeader CTR-weighted visibilityScore — NOT the ungated HealthScoreCard compositeHealthScore. But the locked v3 overview lives on the HOME tab where today only compositeHealthScore is wired (OverviewTab:175-179); the OrientHeader score currently only flows on the strategy tab via strategyData.strategyUx.orient. v3 must decide which score backs the home ring and wire that data path. Picking the visibility score (the better headline) means surfacing orient metrics on the home read, not just strategy.

5. QUIET-MONTH variant has no current home. The v3 quiet-month screen (mockup lines 140-176: stand-card + featured win + 'nothing needs your decision' + in-progress work + dated next check-in) is a first-class state. Today the closest pieces are scattered (AgencyWorkFeed in-progress jobs behind a flag, WinsSurface, the empty-state setup card) and there is no composed 'quiet but reassuring' screen in either version. This is net-new composition.

6. DONE-STATE / 'that's everything' affordance is absent. The v3 mockup ends with an explicit done-state ('✓ That's everything that needs a decision this month — your strategy is on track', line 138). Neither version has a closing 'you're caught up' marker — both just end the scroll. Worth adding as the v3 flow terminator.
