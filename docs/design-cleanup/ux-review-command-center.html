A heuristic review of WorkspaceOverview — the all-workspaces home your operators open first — judged against your own primitives and the four laws of color. The screen does the hard part right: it prioritizes work and deep-links it. What's missing is hierarchy — too many elements claim equal weight, so the eye has nowhere to land. Every fix below uses components you already ship.

What's already right

✓

Needs Attention is prioritized. Items are sorted by severity (churn → requests → approvals → …) and each deep-links to the exact workspace tab. This is the screen's best idea.

✓

Real-time presence. "Online Now" and per-card online banners give operators live context — a genuinely useful, calm signal.

✓

Signature radius is disciplined. StatCards and workspace rows correctly carry the asymmetric corners; nothing else does.

✓

Empty & loading states exist. The zero-workspace welcome and the spinner are designed, not afterthoughts.

01

The findings

F1

The header has five buttons and no primary

High · hierarchy

Prospect, Roadmap, AI Usage, Revenue, and Features sit in the header as five equal secondary buttons — and three are individually colored amber, emerald, and teal. This spends your status colors on navigation: a teal "Features" button reads as the primary action, an amber "AI Usage" button reads as a warning. Per the four laws, those colors mean warning and success — not "this is a nav link."

Now — five colored peers
ProspectRoadmap AI UsageRevenueFeatures
Fix — one neutral cluster
+ New workspace RevenueAI Usage More ▾

RecommendationDemote the five to one neutral secondary cluster (or an overflow "⋯ More" menu), and promote the one thing an operator actually starts here — creating/onboarding a workspace — to the single mint primary. Reserve amber/emerald/teal for status, never chrome.

F2

Six equal StatCards = no headline metric

Medium · hierarchy

The global stat row shows six identical cards — New Requests, Active Requests, Content Pipeline, Approvals, Avg Health, Hours Saved. They're all the same size, so nothing leads, and several duplicate the Needs Attention list directly above (requests, approvals, content all appear in both). The operator reads the same numbers twice with no added meaning.

F3

Your best feature is buried as a plain list

High · hierarchy

Needs Attention is the prioritized, deep-linked worklist — the reason an operator opens this screen. But it renders as an untitled-weight SectionCard sitting between the header and a six-card stat row, with no visual claim on the eye. The most important thing on the page looks like the third-most important.

RecommendationMake Needs Attention the first thing under the header and give it presence — a subtle red/amber left-accent when a P1 exists, a count in the title ("Needs Attention · 6"), and the chevron always visible so it reads as actionable. It earns the spotlight the stat row currently steals.

F4

Workspace rows pack five metric columns of colored micro-text

Medium · density & scannability

Each workspace row crams Health, Requests, Approvals, Content, and SEO Status into one horizontal strip, divided by hairlines, with inline colored counts in every cell ("2 new" red, "3 active" teal, "1 pending" amber…). At three or more workspaces the page becomes a wall of small multicolored numbers — high cognitive load, and the color stops meaning anything because it's everywhere.

Now — 5 columns, color everywhere

Fix — score + one status, rest on demand

RecommendationLead each row with the one number that ranks workspaces (health score, in its band color) plus a single "N need you" pill that rolls up everything actionable. Move the five-column breakdown into the row's expanded/hover state or the workspace detail. Use color only for the rollup status — neutral text for the rest.

F5

Mint is leaking onto data

Low · color law drift

Several non-interactive counts are rendered in text-accent-brand (mint): "3 active" requests, "5 delivered" content, the approvals count. Mint is your action color — when static numbers wear it, everything looks clickable and the genuine CTAs stop standing out. This is the most common drift across the app and the cheapest to fix.

Now — mint on static counts
3 active 5 delivered 3 pending
all read as "click me"
Fix — blue=data, mint=action only
3 active 5 delivered 3 approved
data is blue; success is emerald

RecommendationRecolor static counts to blue (data) or emerald (completed/positive), and let mint appear only on the row's actual click target. A repo-wide sweep for text-accent-brand on non-interactive text would catch most of these in one pass.
