# Post-Parity Owner Closure Contracts

These are the owner-approved invariants for `AUD-D1`–`AUD-D7` and `AUD-B1`. They are narrower than general platform rules and expire only when the closure is fully accepted or superseded by a later owner decision.

## Engine and POV

1. Primary Backing Moves contains exactly the canonical `isActiveRec` set. Every recommendation outside that set remains reachable in one collapsed Operations history home.
2. Weekly Briefing review, recommendation history/un-dismiss/full OV-EMV detail, and SEO Change Impact mount once, under default-collapsed Operations groups. Collapsed groups do not execute their child reads.
3. Strategy POV freshness hashes the exact effective system prompt and user prompt inputs used for generation. Regeneration nonce bypasses cache but is never persisted as the canonical hash.
4. Effective brand voice is injected once. Unused intelligence slices are not assembled.
5. A normal generate or scheduler run never overwrites an operator-edited POV. Changed evidence/voice sets `refreshAvailable`; explicit Regenerate is the only replacement authority.
6. A version-conditional save rejects an AI result when an edit landed while generation was in flight.

## Client Digest

1. The operational monthly digest is current UTC month only.
2. Provider metrics and every operational rollup use the same current-month authority. Mutable all-time stores must not be presented as historical-month facts.
3. No evidence returns `availability: 'no_data'`, deterministic honest copy, and no AI call. A real provider response containing zeroes is still evidence and remains `ready`.
4. Durable historical reports require immutable snapshots and remain a separate backend project.

## Rebuilt Surfaces and Shell

1. Cockpit adds only organic traffic value, content velocity, and overall health. Search clicks/impressions and GA4 users/sessions remain owned by Search & Traffic.
2. Pipeline Published adds compact aggregate impressions and sessions above controls, using the existing authoritative content-performance read model and honest unavailable states.
3. The rebuilt shell exposes the existing HTTP/OpenAI/Webflow/workspace health summary once in a compact footer. It does not duplicate Settings integration controls.
4. Shared Tooltip is above the topmost canonical Modal/Drawer only when its trigger belongs to that overlay. The fix lives in shared overlay machinery, not Asset Manager.
