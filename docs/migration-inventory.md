## Pre-committed acceptance criteria (Gate 5)

**Every Phase 2 PR is "done" when ALL of the following hold:**

- `npm run typecheck && npx vitest run && npx vite build && npx tsx scripts/pr-check.ts` — all green
- `npx tsx scripts/phase2-guard.ts` — exits 0 (no frozen-path violations)
- Playwright visual diff against `tests/playwright/visual/phase2-baseline/` = **zero diffs**, or each diff is explicitly justified in the PR description with snapshot updates committed in the same PR
- No `BUG` / `Critical` review comments open from Devin or manual reviewers
- `Info` / `Minor` / `Suggestion` comments are acknowledged in a follow-up list but **do not block merge**
- If the PR changes a primitive consumption pattern that affects the per-component color map: `BRAND_DESIGN_LANGUAGE.md` § 4 updated (consolidated by integrator, not the worker)
- Phase 2 worker branches **never** modify any path in §5 of the kickoff doc (frozen primitive sources, `DESIGN_SYSTEM.md §1–§20`, infra/config, server/, shared/types/)
- The PR's first commit is `chore: apply codemods to <domain>` (auto-generated diff). Manual residual fixes go in subsequent commits.

## Worker dispatch checklist

Before dispatching a worker for any task in this inventory, the dispatch prompt
MUST include:

1. The parent Phase 5 plan link
2. The kickoff doc link (this is non-optional context)
3. **This inventory doc link** with the worker's specific task ID highlighted
4. The exact file list from the worker's task (copy-pasted into the prompt — workers
   do not "discover" their files)
5. The §5 DO NOT TOUCH list (always verbatim)
6. The §6 canonical playbook (always verbatim)
7. The pre-committed acceptance criteria (top of this doc)
8. Branch name (`feat/design-system-phase5-2.X-<slug>`) and PR target (`staging`)
9. Reminder to run `npx tsx scripts/phase2-guard.ts` before every push
10. Reminder to capture pre-edit screenshots and run Playwright visual diff before opening PR

Workers run in **parallel batches of 3–4** (review bandwidth caps at that — do not
dispatch all 12 at once). Suggested batch order:

- **Batch 1:** 2.1a, 2.1b, 2.1c (admin analytics — heaviest single domain, get it shipped first)
- **Batch 2:** 2.2, 2.3a (admin content + audit — independent, no shared files)
- **Batch 3:** 2.3b, 2.4a (admin ops + client core — keep AdminChat purple intact)
- **Batch 4:** 2.4b, 2.4c (client insights + onboarding — both have Law 04 zero-purple guards)
- **Batch 5:** 2.5a, 2.5b (brand/schema + revenue/settings)
- **Batch 6:** 2.6 (platform shell — ship last so prior batches' baseline screenshots are stable)

Re-baseline Playwright after each merge to staging — the next batch diffs against
the most recent state.

## Integrator monitoring duties

While workers run (kickoff §3, "During dispatch"):

- Every 2–3 minutes: `git fetch --all` then `git diff origin/feat/...phase5-2.X...origin/feat/...phase5-2.Y -- <shared-paths>` for any shared touched files
- Pause and reconcile any worker that drifts into another's ownership
- Consolidate doc updates: workers emit "doc change suggestions" back; integrator commits to `chore/phase2-doc-consolidation` (a follow-up branch off staging) — never to a worker branch

Post-dispatch, pre-PR (kickoff §3, "Post-dispatch"):

- Cross-PR consistency grep across all 12 branches:
  - `cn(` usage (Three Laws-of-className-merge — playbook §6.1) — every dynamic className must use `cn()`, never template literal or `[].filter().join()`
  - Raw hex colors in JSX (`grep -rE "#[0-9a-fA-F]{3,6}" <branch-files>`)
  - `text-zinc-/bg-zinc-/border-zinc-` raw uses against the residue threshold (post-codemod sites only — pre-existing zinc is Phase 3 scope)
- Run Playwright visual diff per branch — only zero-diff branches open PRs
